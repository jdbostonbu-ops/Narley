# SPRINT-USER-STORIES.md

# Narley Sprint User Stories Library

**Project:** Narley  
**Document Version:** 3.0.0  
**Status:** Living Document

## Purpose

This document is a reusable library of user stories for Narley. Each Friday, select the stories that match the sprint goal and copy them into your sprint planning template.

---

## Reader Authentication

- As a Reader, I want to create an account so that I can save resources and personalize my experience.
- As a Reader, I want to sign in with Google so that I can access Narley quickly.
- As a Reader, I want to verify my email so that my account is secure.

## Map & Search

- As a Reader, I want to see nearby resources on a map so that I can quickly find help.
- As a Reader, I want to search by city or ZIP Code so that I can locate resources in another area.
- As a Reader, I want to recenter the map on my current location so that I can quickly find nearby services.

## Resource Pins

- As a Reader, I want category icons on map pins so that I can identify resources at a glance.
- As a Reader, I want the map pin and resource card to use the same icon so that I know they represent the same resource.
- As a Reader, I want accessible map pins so that I can recognize resources regardless of reading ability.

## Resource Cards & Modal

- As a Reader, I want to tap either a map pin or a resource card and see the same resource details so that my experience is consistent.
- As a Reader, I want complete resource information in the detail modal so that I can decide whether the resource meets my needs.

## Saved Resources

- As a Reader, I want to save resources so that I can find them again later.
- As a Reader, I want reminders for saved resources so that I remember important dates.

## Directions

- As a Reader, I want in-app route previews for walking, driving, transit, and bicycling so that I can choose the best way to travel.
- As a Reader, I want to open my preferred maps application if in-app routing is unavailable.

## Community Reporting

- As a Reader, I want to report incorrect resource information so that Providers can review it.
- As a Reader, I want structured report categories so that reporting is simple and consistent.
- As a Reader, I want confirmation after submitting a report so that I know it was received.

## Provider Resource Management

- As a Provider, I want to create resources so that Readers can discover available services.
- As a Provider, I want to edit resources so that published information stays current.
- As a Provider, I want to pause or close resources so that Readers receive accurate information.
- As a Provider, I want confirmation before destructive actions so that accidental changes are prevented.

## Community Intelligence

- As a Provider, I want to review Reader reports so that I can verify resource information.
- As a Provider, I want to prioritize review items so that I can address the most important issues first.
- As a Provider, I want an audit history so that resource changes remain transparent.

## AI Verification

- As a Provider, I want AI to gather publicly available evidence so that I spend less time researching.
- As a Provider, I want AI confidence scores so that I can prioritize my review.
- As a Provider, I want AI recommendations without automatic changes so that I remain in control of published information.

## Accessibility

- As a Reader, I want large touch targets so that the application is easier to use.
- As a Reader, I want screen reader support so that Narley is accessible.
- As a Reader, I want icons and text together so that information is never conveyed by color alone.

## Example Friday Sprint

**Today's Goal:** Build the configurable map pin system.

### Must Do

- As a Reader, I want category icons on map pins so that I can identify resources quickly.
- As a Reader, I want the map pin and resource card to display the same icon so that I know they represent the same resource.

### Nice to Have

- As a Reader, I want accessible map pins so that I can recognize resources regardless of language.

### Stretch

- As a Developer, I want the pin system to be configuration-driven so that new categories can be added without changing rendering logic.
- As a Reader, I want to use Narley in my preferred language so that I can understand the interface.
- As a Provider, I want my language preference remembered so that I don't need to change it every time I sign in.
