# Purpose

This document records significant changes made during the development of Narley Version 3.

It provides a historical record of architectural decisions, feature additions, bug fixes, documentation updates, and implementation milestones.

Unlike the Roadmap, which describes planned work, the Changelog documents completed work.

---

# Changelog Format

Each entry should include:

- Version
- Date
- Type of Change
- Description
- Related Documents (if applicable)

Example:

```text
Version: 3.0.2

Date:
2026-08-14

Type:
Feature

Description:
Added Community Intelligence Dashboard to Provider application.

Related Documents:
PROVIDER-APP-SPEC.md
COMMUNITY-INTELLIGENCE.md
```

# Maintenance

The changelog should be updated whenever:

- A feature is completed.
- A significant bug is fixed.
- Documentation changes materially affect implementation.
- Architecture changes.
- Security changes.
- Accessibility changes.
- CPA-approved changes are implemented.

Small cosmetic changes do not require changelog entries unless they affect user experience.

---

# Guiding Principle

The Version 3 Changelog is the official historical record of Narley's development.

Every significant change should be documented so future developers, AI coding assistants, and project stakeholders can understand how the project evolved over time.
